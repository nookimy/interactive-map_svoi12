Инструкция по заполнению данных для locationData

Этот файл содержит данные по участкам, которые используются для отображения информации на карте и в балуне.

Общая структура объекта locationData

Каждый участок в locationData задаётся как объект, где ключ — это id элемента SVG, а значение — объект с параметрами:

locationData = {
  'id_элемента': {
    name: 'Название участка или номер',
    status: STATUS_FREE,        // Статус участка (см. ниже)
    area: 1234.56,             // Площадь участка в м² (число)
    price: 12345678,           // Цена участка в рублях (число)
    houseArea: 150,            // Площадь дома в м² (число, необязательно)
    housePrice: 23500000,      // Цена дома в рублях (число, необязательно)
    img: 'plan-image1.jpg'     // Имя файла изображения дома (строка, необязательно)
  },
  // другие участки...
}

Пояснения к параметрам

  • name — название или номер участка. Обязательное для отображения.
  • status — статус участка. Используйте одну из предопределённых переменных статуса (ниже). Если не указано — будет присвоен статус STATUS_FREE.
  • area — площадь участка в квадратных метрах (число). Отображается с округлением вниз и с разделением тысяч пробелами.
  • price — цена участка в рублях (число). Отображается с разделением тысяч пробелами и знаком ₽.
  • houseArea — площадь дома в м² (число). Необязательный параметр. Если не указан — используется значение по умолчанию.
  • housePrice — цена дома в рублях (число). Необязательный параметр. Если не указан — используется значение по умолчанию.
  • img — имя файла с изображением дома (например, plan-image1.jpg). Необязательный параметр. Изображения должны находиться в папке img/ вашего проекта. Если указано, изображение будет показано в балуне.

Статусы участка

Чтобы избежать ошибок в написании, используйте в параметре status переменные из кода. Ниже перечислены предопределённые переменные, которые доступны в вашем скрипте:

const STATUS_FREE = 'Свободный участок';
const STATUS_HOUSE_READY = 'Участок с готовым домом';
const STATUS_HOUSE_BUILDING = 'Участок со строящимся домом';
const STATUS_RESERVED = 'Забронированный участок';
const STATUS_SOLD = 'Проданный участок';

Пример правильного указания статуса в locationData:

locationData = {
  'path12345': {
    name: 'Участок 12',
    status: STATUS_HOUSE_READY,
    area: 1800,
    price: 9500000,
    houseArea: 160,
    housePrice: 22000000,
    img: 'plan-image12.jpg'
  }
}

Пример полного объекта locationData

const locationData = {
  'path62461': {
    name: 'Участок 3665',
    status: STATUS_FREE,
    area: 1669,
    price: 8900000,
    houseArea: 150,
    housePrice: 23500000,
    img: 'plan-image1.jpg'
  },
  'path62465': {
    name: 'Участок 3277',
    status: STATUS_SOLD,
    area: 1132,
    price: 6000000,
    houseArea: 140,
    housePrice: 21000000
  },
  'p1-1': {
    name: '1.1',
    status: STATUS_HOUSE_BUILDING,
    area: 1532.24,
    price: 7200000
    // houseArea и housePrice будут использованы значения по умолчанию
  }
};

Советы

  • Всегда используйте переменные для status, чтобы избежать опечаток.
  • Если параметр не нужен или неизвестен, его можно опустить.
  • Цены и площади указывайте в числовом формате (без пробелов и дополнительных символов).
  • Все изображения должны лежать в папке img/ вашего проекта, и указывайте только имя файла в параметре img.
